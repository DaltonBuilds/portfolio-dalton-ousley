# AWS Infrastructure - Terraform Configuration

This directory contains the Terraform configuration for the serverless lead capture system.

## Prerequisites

- [Terraform](https://www.terraform.io/downloads) >= 1.5.0
- [AWS CLI](https://aws.amazon.com/cli/) configured with appropriate credentials
- Node.js >= 20.0.0 (for Lambda builds)
- [Resend](https://resend.com) account with verified domain
- [Cloudflare Turnstile](https://www.cloudflare.com/products/turnstile/) configured

## Architecture

```
Internet → CloudFront/Cloudflare Pages (Frontend)
             ↓
         API Gateway (with WAF)
             ↓
         Lambda (Lead Processor)
             ↓
         ┌──────────────┬──────────────┐
         ↓              ↓              ↓
     DynamoDB      EventBridge    CloudWatch
                       ↓
                  Lambda (Email Notifier)
                       ↓
                   Resend API
```

## Infrastructure Components

- **DynamoDB**: Lead storage with TTL for PII hygiene
- **Lambda Functions**:
  - Lead Processor: Validates and stores leads
  - Email Notifier: Sends notifications via Resend
- **API Gateway**: HTTP API endpoint
- **EventBridge**: Event-driven architecture for lead notifications
- **Secrets Manager**: Secure storage for API keys and secrets
- **WAF**: Protection against common attacks and rate limiting
- **CloudWatch**: Logging, monitoring, and alarms

## Quick Start

### 1. Build Lambda Functions

```bash
cd /Users/daltonousley/dev/portfolio-dalton-ousley
./scripts/build-lambdas.sh
```

### 2. Configure Variables

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your values
```

Required variables:
- `owner_email` - Your email for infrastructure tagging
- `notification_email` - Where to send lead notifications
- `notification_from_email` - Sender email (must be verified in Resend)

### 3. Initialize Terraform

```bash
terraform init
```

### 4. Plan Deployment

```bash
terraform plan -out=tfplan
```

Review the plan carefully before applying.

### 5. Apply Configuration

```bash
terraform apply tfplan
```

### 6. Populate Secrets

After terraform apply, you must manually populate the Turnstile and Resend secrets:

```bash
# Cloudflare Turnstile Secret
aws secretsmanager put-secret-value \
  --secret-id portfolio/turnstile-secret \
  --secret-string "YOUR_TURNSTILE_SECRET_KEY"

# Resend API Key
aws secretsmanager put-secret-value \
  --secret-id portfolio/resend-api-key \
  --secret-string "YOUR_RESEND_API_KEY"
```

The HMAC server secret is automatically generated by Terraform.

### 7. Update Frontend Environment Variables

Copy the API Gateway URL from Terraform outputs:

```bash
terraform output api_gateway_url
```

Add to `.env.local` and Cloudflare Pages environment variables:

```bash
NEXT_PUBLIC_API_GATEWAY_URL=<from-terraform-output>
NEXT_PUBLIC_HMAC_CLIENT_SECRET=<generate-with-openssl-rand-hex-32>
NEXT_PUBLIC_TURNSTILE_SITE_KEY=<from-cloudflare-dashboard>
```

## Common Operations

### View Outputs

```bash
terraform output
```

### View Logs

```bash
# Lead Processor logs
aws logs tail /aws/lambda/portfolio-leads-prod-lead-processor --follow

# Email Notifier logs
aws logs tail /aws/lambda/portfolio-leads-prod-email-notifier --follow

# API Gateway logs
aws logs tail /aws/apigateway/portfolio-leads-prod --follow
```

### Query Leads from DynamoDB

```bash
# Get all leads
aws dynamodb scan \
  --table-name portfolio-leads-prod-table

# Get leads by date range (using GSI)
aws dynamodb query \
  --table-name portfolio-leads-prod-table \
  --index-name createdAt-index \
  --key-condition-expression "#type = :type AND createdAt BETWEEN :start AND :end" \
  --expression-attribute-names '{"#type":"type"}' \
  --expression-attribute-values '{":type":{"S":"LEAD"},":start":{"N":"1609459200000"},":end":{"N":"1640995200000"}}'
```

### Test API Endpoint

```bash
# Get API URL
API_URL=$(terraform output -raw api_gateway_url)

# Test with curl (requires valid HMAC signature)
curl -X POST $API_URL \
  -H "Content-Type: application/json" \
  -H "X-HMAC-Signature: <signature>" \
  -H "X-HMAC-Timestamp: <timestamp>" \
  -H "X-Idempotency-Key: <uuid>" \
  -H "X-Turnstile-Token: <token>" \
  -d '{"name":"Test","email":"test@example.com","message":"Test message","turnstileToken":"<token>"}'
```

### Update Lambda Functions

After making changes to Lambda code:

```bash
# Rebuild
./scripts/build-lambdas.sh

# Reapply Terraform
cd terraform
terraform apply
```

### Rotate Secrets

```bash
# Generate new HMAC server secret
NEW_SECRET=$(openssl rand -hex 32)

# Update in Secrets Manager
aws secretsmanager update-secret \
  --secret-id portfolio/hmac-server-secret \
  --secret-string "$NEW_SECRET"

# Lambda will pick up new secret on next invocation (due to caching)
```

## Monitoring

### CloudWatch Dashboards

Create a dashboard to monitor key metrics:

```bash
aws cloudwatch put-dashboard \
  --dashboard-name portfolio-leads \
  --dashboard-body file://dashboard.json
```

### Alarms

The following CloudWatch alarms are automatically created:

- **DynamoDB Throttling**: Alerts on throttled requests
- **Lambda Errors**: Alerts on function errors
- **Lambda Duration**: Alerts on slow executions
- **API Gateway 4XX/5XX**: Alerts on API errors
- **WAF Blocked Requests**: Alerts on high block rates
- **DLQ Messages**: Alerts when email notifications fail

## Troubleshooting

### Lambda Function Errors

1. Check CloudWatch Logs:
   ```bash
   aws logs tail /aws/lambda/portfolio-leads-prod-lead-processor --follow
   ```

2. Check Lambda environment variables are correct

3. Verify secrets are populated in Secrets Manager

### API Gateway 403 Errors

- Check WAF rules aren't blocking legitimate requests
- Verify CORS configuration matches frontend origin

### Email Notifications Not Sending

1. Check email notifier Lambda logs
2. Verify Resend API key is correct
3. Check sender email is verified in Resend
4. Look for messages in DLQ:
   ```bash
   aws sqs receive-message \
     --queue-url <dlq-url>
   ```

### HMAC Verification Failures

- Ensure frontend and backend use compatible signature algorithms
- Check timestamp is within 5-minute window
- Verify secret is correctly populated

## Security Best Practices

1. ✅ Secrets stored in Secrets Manager (not code)
2. ✅ Least-privilege IAM roles
3. ✅ WAF enabled for DDoS protection
4. ✅ HMAC request signing
5. ✅ Cloudflare Turnstile bot protection
6. ✅ PII hygiene with TTL (18 months)
7. ✅ No PII in CloudWatch logs
8. ✅ HTTPS only (enforced by API Gateway)
9. ✅ Rate limiting (WAF + API Gateway)
10. ✅ Input validation (Zod schemas)

## Cleanup

To destroy all infrastructure:

```bash
terraform destroy
```

**Warning**: This will permanently delete:
- All leads in DynamoDB
- CloudWatch logs
- Secrets (after 7-day recovery window)

Ensure you've exported any data you need before destroying.
