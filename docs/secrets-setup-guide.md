# Secrets Setup Guide - AWS Secrets Manager

**Important:** This guide shows you how to securely store secrets in AWS Secrets Manager after deploying the infrastructure.

---

## Overview

Your lead capture system uses **three secrets** stored in AWS Secrets Manager:

1. **Turnstile Secret** - Cloudflare bot protection (you provide this)
2. **Resend API Key** - Email notifications (you provide this)  
3. **HMAC Server Secret** - Request verification (auto-generated by Terraform)

## Step 1: Deploy Infrastructure First

The Secrets Manager resources must exist before you can populate them:

```bash
cd terraform
terraform apply
```

Type `yes` when prompted. This will create all AWS resources including the empty secret placeholders.

---

## Step 2: Add Turnstile Secret

After `terraform apply` completes, add your Cloudflare Turnstile **secret key**:

### Method 1: AWS CLI (Recommended)

```bash
aws secretsmanager put-secret-value \
  --secret-id portfolio/turnstile-secret \
  --secret-string "YOUR_TURNSTILE_SECRET_KEY_HERE" \
  --region us-east-1 \
  --profile AdministratorAccess-123456789012
```

### Method 2: AWS Console

1. Go to: https://console.aws.amazon.com/secretsmanager/
2. Select region: **us-east-1**
3. Find: `portfolio/turnstile-secret`
4. Click "Retrieve secret value"
5. Click "Edit"
6. Paste your Turnstile **secret key** (NOT the site key)
7. Click "Save"

---

## Step 3: Add Resend API Key

Get your Resend API key from: https://resend.com/api-keys

### Method 1: AWS CLI (Recommended)

```bash
aws secretsmanager put-secret-value \
  --secret-id portfolio/resend-api-key \
  --secret-string "re_YOUR_RESEND_API_KEY_HERE" \
  --region us-east-1 \
  --profile AdministratorAccess-123456789012
```

### Method 2: AWS Console

1. Go to: https://console.aws.amazon.com/secretsmanager/
2. Select region: **us-east-1**
3. Find: `portfolio/resend-api-key`
4. Click "Retrieve secret value"
5. Click "Edit"
6. Paste your Resend API key (starts with `re_`)
7. Click "Save"

## Step 4: Verify Secrets (Optional)

Check that secrets are properly stored:

```bash
# Verify Turnstile secret exists
aws secretsmanager get-secret-value \
  --secret-id portfolio/turnstile-secret \
  --region us-east-1 \
  --profile AdministratorAccess-123456789012 \
  --query 'SecretString' \
  --output text

# Verify Resend API key exists

  --secret-id portfolio/resend-api-key \
  --region us-east-1 \
  --profile AdministratorAccess-123456789012 \
  --query 'SecretString' \

# Verify HMAC server secret (auto-generated by Terraform)

  --region us-east-1 \
  --profile AdministratorAccess-123456789012 \

  --output text
```

## Important: What Goes Where?

### Frontend (.env.local) - PUBLIC
These are **exposed to the browser** and safe to be public:

```bash
# Site Key (PUBLIC - from Cloudflare Turnstile)
NEXT_PUBLIC_TURNSTILE_SITE_KEY=1x00000000000000000000AA

# API Gateway URL (from Terraform outputs)
NEXT_PUBLIC_API_GATEWAY_URL=https://your-api-id.execute-api.us-east-1.amazonaws.com/leads

# HMAC Client Secret (generate separately - different from server secret)
NEXT_PUBLIC_HMAC_CLIENT_SECRET=$(openssl rand -hex 32)
```

### AWS Secrets Manager - PRIVATE
These are **server-side secrets** that NEVER go in frontend code:

```bash
portfolio/turnstile-secret      → Turnstile SECRET KEY (from Cloudflare)
portfolio/resend-api-key        → Resend API Key (from Resend dashboard)
portfolio/hmac-server-secret    → Auto-generated by Terraform
```

## Security Notes

### ✅ Good Practices

1. **Turnstile Site Key** → Goes in `.env.local` (public, browser-visible)
2. **Turnstile Secret Key** → Goes in AWS Secrets Manager (private, server-only)
3. **Resend API Key** → Goes in AWS Secrets Manager (private, server-only)
4. **HMAC Server Secret** → Auto-generated, stays in AWS Secrets Manager
5. **HMAC Client Secret** → Different secret, goes in `.env.local` (public)

### ❌ Never Do This

- ❌ Don't put Turnstile secret key in `.env.local`
- ❌ Don't put Resend API key in `.env.local`
- ❌ Don't commit secrets to git
- ❌ Don't use the same HMAC secret for client and server

### 🔐 How It Works

**Request Flow:**
1. Frontend signs request with `NEXT_PUBLIC_HMAC_CLIENT_SECRET`
2. Frontend includes Turnstile token from widget
3. Backend verifies BOTH signatures:
   - Client signature (using `NEXT_PUBLIC_HMAC_CLIENT_SECRET`)
   - Server signature (using `portfolio/hmac-server-secret` from Secrets Manager)
4. Backend verifies Turnstile token with Cloudflare using secret key from Secrets Manager
5. Backend sends email via Resend using API key from Secrets Manager

## Troubleshooting

### "AccessDeniedException: User is not authorized"

Your IAM user needs permission to access Secrets Manager. Check that your SSO role has `secretsmanager:GetSecretValue` and `secretsmanager:PutSecretValue` permissions.

**Fix:** Use your SSO profile with admin access:
```bash
aws secretsmanager put-secret-value \
  --profile AdministratorAccess-123456789012 \
  ... rest of command
```

### "ResourceNotFoundException"

The secret doesn't exist yet. Make sure you've run `terraform apply` first to create the Secrets Manager resources.

### "Lambda function fails with 'Secret not found'"

The secret exists but has no value. Use `put-secret-value` to populate it (see Step 2 and 3 above).

## Rotating Secrets

### Recommended Schedule
- **Turnstile Secret:** Rotate every 90 days
- **Resend API Key:** Rotate every 90 days
- **HMAC Secrets:** Rotate every 90 days

### How to Rotate

1. Generate new secret in Cloudflare/Resend
2. Update in AWS Secrets Manager:
   ```bash
   aws secretsmanager put-secret-value \
     --secret-id portfolio/turnstile-secret \
     --secret-string "NEW_SECRET_HERE" \
     --region us-east-1 \
     --profile AdministratorAccess-123456789012
   ```
3. Lambda automatically picks up new value on next cold start
4. No code changes needed!
